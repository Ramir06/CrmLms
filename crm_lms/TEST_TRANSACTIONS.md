# Инструкция по тестированию автоматических транзакций

## Что должно работать:
1. При создании платежа студента (с указанием paid_at) автоматически создается транзакция в бухгалтерии
2. При изменении статуса зарплаты на "Выплачено" автоматически создается транзакция расхода

## Как протестировать:

### 1. Перезапустите Django сервер
После добавления отладочных print() в сигналы, перезапустите сервер чтобы изменения применились.

### 2. Создайте тестовый платеж
1. Зайдите в админку -> Оплаты -> Добавить оплату
2. Заполните форму:
   - Студент: выберите любого студента
   - Курс: выберите курс
   - Сумма: укажите сумму
   - Дата оплаты: укажите сегодняшнюю дату
   - Способ оплаты: выберите способ
3. Нажмите "Сохранить"

### 3. Проверьте консоль сервера
В консоли должны появиться сообщения:
```
=== SIGNAL FIRED: Payment 123, created=True, paid_at=2025-03-30 ===
Payment has paid_at: 2025-03-30
Creating new transaction for payment 123
=== CREATING PAYMENT TRANSACTION ===
Payment ID: 123
Student: Иван Иванов
Student organization: Организация 1
...
Transaction created: 456 - 10000.00
```

### 4. Проверьте бухгалтерию
1. Зайдите в Бухгалтерия -> Транзакции
2. Должна появиться новая транзакция с типом "Доход"
3. В описании будет что-то вроде: "Оплата курса: Название курса - Имя студента"
4. Поле "Авто" должно быть "Да"

### 5. Протестируйте зарплаты
1. Зайдите в Зарплаты -> Добавить начисление
2. Выберите ментора, укажите сумму
3. Сохраните со статусом "Ожидает"
4. Затем редактируйте и поменяйте статус на "Выплачено"
5. В консоли должны появиться сообщения о создании транзакции расхода

## Если транзакции не создаются:

### Возможные проблемы:
1. **Сигналы не зарегистрированы** - проверьте apps.py в finance и organizations
2. **Нет организации у студента/ментора** - проверьте что у пользователей есть организация
3. **Ошибка в логике** - посмотрите сообщения об ошибках в консоли

### Как проверить:
1. В Django shell выполните:
```python
from apps.payments.models import Payment
from apps.finance.models import FinanceTransaction

# Создайте тестовый платеж напрямую
payment = Payment.objects.create(
    student=student_obj,
    course=course_obj,
    amount=10000,
    paid_at="2025-03-30",
    created_by=user_obj
)

# Проверьте появилась ли транзакция
tx = FinanceTransaction.objects.filter(related_entity_type='payment', related_entity_id=payment.id).first()
print(tx)
```

## Команда для исправления данных:
Если есть старые платежи без транзакций:
```bash
python manage.py fix_finance_organization
```

Эта команда создаст недостающие категории/счета и исправит организации.
