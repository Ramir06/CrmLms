# Ограничение доступа к посещаемости для заменяющих менторов

## Проблема
Ранее заменяющие менторы имели доступ к посещаемости всех уроков курса, даже в те дни, когда они не заменяли основного ментора.

## Решение
Реализована система, которая ограничивает доступ к посещаемости только тем дням, когда ментор официально назначен заменяющим.

## Основные изменения

### 1. Обновлен `apps/attendance/views.py`
- Добавлена функция `get_mentor_course_with_substitute_access()` для проверки прав доступа
- Функция `attendance_table()` фильтрует уроки только по дням замен для заменяющих менторов
- Добавлена проверка прав в `save_attendance_bulk()` для предотвращения отметки посещаемости в недоступные дни
- В контекст добавлен флаг `is_substitute_mentor` для определения типа пользователя

### 2. Обновлен шаблон `templates/mentor/attendance/table.html`
- Добавлено информационное сообщение для заменяющих менторов
- Ячейки посещаемости недоступных для отметки дней помечаются классом `disabled`
- Заблокированные ячейки не показывают hover-меню и не реагируют на клики
- Функция "выбрать все" работает только с доступными ячейками
- Добавлены стили для визуального обозначения недоступных дней

### 3. Использованы существующие миксины
- Применяется функция `can_mark_attendance()` из `apps/core/mixins_substitute.py`
- Используется функция `check_substitute_access()` для проверки общего доступа

## Как это работает

1. **Основной ментор**: имеет полный доступ к посещаемости всех уроков курса
2. **Заменяющий ментор**: видит только те уроки, на которые назначен замена
3. **Администратор**: имеет полный доступ ко всем функциям

## Визуальные индикаторы

- Для заменяющего ментора показывается синяя информационная панель
- Недоступные для отметки дни имеют серый фон и пониженную прозрачность
- При наведении на недоступные ячейки курсор меняется на "not allowed"
- Недоступные ячейки не показывают меню с опциями отметки

## Безопасность

- Проверка прав выполняется как на уровне отображения, так и на уровне сохранения
- Попытка отметки посещаемости в недоступный день блокируется с сообщением об ошибке
- Система предотвращает прямые запросы к API в обход интерфейса

## URL доступа

- Основной ментор: `/lms/mentor/course/{course_id}/attendance/`
- Заменяющий ментор: `/lms/mentor/course/{course_id}/attendance/` (показывает только доступные дни)
- Список замен: `/lms/admin/mentors/substitute/`
